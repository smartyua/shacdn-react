# Skill: api-design

## When to use
Changing component public props or MCP tool contracts

## Inputs
Proposed contract

## Steps
1. Compare with existing component props / MCP tools
2. Prefer backward-compatible changes
3. Document breaking changes explicitly
4. Update COMPONENTS_AI_REFERENCE / MCP docs

## Required checks
- types compile
- docs updated

## Expected result
Stable contract

## Forbidden
- Silent breaking changes

## Done when
Contract + docs aligned
