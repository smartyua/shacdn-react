# Skill: code-review

## When to use
Reviewing a diff before claiming completion

## Inputs
Diff; original requirements

## Steps
1. Verify correctness vs requirements
2. Check tests, types, security, a11y for UI
3. Look for regressions and unnecessary complexity
4. Confirm docs/memory updates when needed

## Required checks
- adversarial pass attempted
- requirements re-read

## Expected result
Review notes with severity

## Forbidden
- Rubber-stamping author summary

## Done when
Blocking issues resolved or explicitly listed
