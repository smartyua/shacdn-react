# Skill: incident-analysis

## When to use
Production-like incident or severe demo breakage

## Inputs
Timeline; symptoms

## Steps
1. Establish timeline
2. Identify blast radius
3. Find root cause
4. Document in failures memory
5. Propose prevention

## Required checks
- root cause named

## Expected result
Incident note

## Forbidden
- Blame without evidence

## Done when
Prevention action listed
