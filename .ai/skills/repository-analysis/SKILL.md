# Skill: repository-analysis

## When to use
Orienting in an unfamiliar area of the repo or starting a large task

## Inputs
Task description; optional paths

## Steps
1. Read .ai/knowledge/project-overview.md and architecture.md
2. Read .ai/memory/semantic/project-facts.md
3. Search existing components/utilities before proposing new ones
4. Check docs/AI_AGENT_GUIDE.md and SHADCN_PARITY_MATRIX.md for UI work
5. Optionally run npm run ai:rag -- <query>
6. Summarize findings with file paths

## Required checks
- Facts cited from real files
- No invented APIs

## Expected result
Short orientation memo with paths and risks

## Forbidden
- Rewriting unrelated modules
- Skipping existing docs

## Done when
Orientation memo written; next step clear
